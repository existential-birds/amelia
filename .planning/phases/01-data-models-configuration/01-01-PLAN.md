<feature>
  <name>PR Auto-Fix Data Models</name>
  <files>amelia/core/types.py, tests/unit/core/test_pr_autofix_models.py</files>
  <behavior>
    ## AggressivenessLevel (DATA-04)
    - AggressivenessLevel is an IntEnum with CRITICAL=1, STANDARD=2, THOROUGH=3
    - Supports ordering: CRITICAL < STANDARD < THOROUGH
    - Supports threshold comparisons: `level >= AggressivenessLevel.STANDARD`

    ## PRSummary (DATA-01)
    - Frozen Pydantic model with: number (int), title (str), head_branch (str), author (str), updated_at (datetime)
    - All fields required, no defaults
    - Immutable: assignment raises error

    ## PRReviewComment (DATA-02)
    - Frozen Pydantic model with required fields: id (int), body (str), author (str), created_at (datetime)
    - Optional inline fields: path (str|None), line (int|None), diff_hunk (str|None) -- all default None
    - Optional thread fields: in_reply_to_id (int|None), thread_id (str|None), node_id (str|None) -- all default None
    - Optional context: pr_number (int|None) -- default None (Claude's discretion: include for self-contained context)
    - General comments have path=None, line=None, diff_hunk=None
    - Inline comments have all three populated

    ## PRAutoFixConfig (DATA-03, CONF-01)
    - Frozen Pydantic model with defaults for all fields:
      - aggressiveness: AggressivenessLevel = STANDARD
      - poll_interval: int = 60 (ge=10, le=3600)
      - auto_resolve: bool = True
      - max_iterations: int = 3 (ge=1, le=10)
      - commit_prefix: str = "fix(review):"
    - Validation rejects poll_interval < 10 or > 3600
    - Validation rejects max_iterations < 1 or > 10

    ## Profile integration (CONF-02, CONF-03)
    - Profile gains `pr_autofix: PRAutoFixConfig | None = None`
    - Profile with pr_autofix=None means auto-fix disabled
    - Profile with pr_autofix=PRAutoFixConfig() means enabled with defaults
    - PRAutoFixConfig supports override via model_copy: `config.model_copy(update={"aggressiveness": AggressivenessLevel.THOROUGH})`

    ## Serialization
    - AggressivenessLevel serializes to string name (not integer) in JSON for API readability
    - All models round-trip through model_dump() / model_validate()
  </behavior>
