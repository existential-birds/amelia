    """Async service for GitHub PR operations via gh CLI."""

**Private helper: `_run_gh`**
- Uses `asyncio.create_subprocess_exec("gh", *args, ...)` with stdout/stderr PIPE and `cwd=self._repo_root`
- Timeout parameter (default 30s) using `asyncio.wait_for`
- On timeout: kill process, raise `ValueError(f"gh command timed out after {timeout}s: gh {' '.join(args)}")`
- On non-zero exit: raise `ValueError(f"gh command failed: {stderr}")`
- Returns stdout string

**Method: `fetch_review_comments(pr_number: int, ignore_authors: list[str] | None = None) -> list[PRReviewComment]`** (GHAPI-01, GHAPI-05)
- Two-step approach per research:
  1. REST: `gh api --paginate /repos/{owner}/{repo}/pulls/{pr_number}/comments` -- parse JSON array
  2. GraphQL: Query `repository.pullRequest.reviewThreads.nodes` to get thread IDs and `isResolved` status. Use `first: 100`. Log warning if `pageInfo.hasNextPage` is true.
- Map REST comment `id` to GraphQL thread ID via `comments.nodes.databaseId` matching
- Filter out resolved threads (isResolved=true)
- Filter out self-comments (body contains `_Amelia (automated fix)_`) -- always checked
- Filter out ignore_authors (exact match) -- if list provided
- Build PRReviewComment instances with all fields including `thread_id` and `node_id` from GraphQL data
- Set `pr_number` on each comment

**Method: `list_open_prs() -> list[PRSummary]`** (GHAPI-02)
- `gh pr list --json number,title,headRefName,author,updatedAt --state open --limit 100`
- Parse JSON, map `headRefName` -> `head_branch`, `author["login"]` -> `author`
- Return list of PRSummary

**Method: `resolve_thread(thread_node_id: str) -> None`** (GHAPI-03)
- GraphQL mutation: `resolveReviewThread(input: {threadId: $threadId})`
- Via `gh api graphql -f query=... -f threadId=...`
- Raise ValueError on failure

**Method: `reply_to_comment(pr_number: int, comment_id: int, body: str, in_reply_to_id: int | None = None) -> None`** (GHAPI-04)
- Append Amelia footer: `f"{body}\n\n---\n_Amelia (automated fix)_"`
- Use parent comment ID if `in_reply_to_id` is set (per Pitfall 7: reply endpoint requires top-level comment ID)
- `gh api --method POST /repos/{owner}/{repo}/pulls/{pr_number}/comments/{target_id}/replies -f body=...`

**Constant: `AMELIA_FOOTER = "_Amelia (automated fix)_"`**

**Method: `_should_skip_comment(comment_body: str, comment_author: str, ignore_authors: list[str]) -> bool`** (GHAPI-05)
- Return True if `AMELIA_FOOTER in comment_body`
- Return True if `comment_author in ignore_authors`
- Return False otherwise
