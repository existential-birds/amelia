  <behavior>
    classify_node:
    - Calls extract_config_params(config) to get event_bus, workflow_id, profile
    - Gets a driver instance for the classifier (use get_driver with profile's agent config)
    - Calls filter_comments(state.comments, {}, state.autofix_config.max_iterations) for pre-filtering (pass empty dict for all_thread_comments since we don't have thread context at this level -- threads are pre-filtered by the caller)
    - Calls classify_comments(filtered, driver, state.autofix_config) for LLM classification
    - Calls group_comments_by_file(state.comments, classifications) for file grouping
    - Returns dict with classified_comments (list of CommentClassification values), file_groups (dict mapping file_path to list of comment IDs)
    - If no comments in state, returns empty results without calling LLM

    develop_node:
    - Calls extract_config_params(config) to get profile
    - Iterates over state.file_groups items (Python loop, not LangGraph conditional)
    - For each file group: builds a Developer goal string with comment body, file path, line number, diff hunk, PR metadata (pr_number, head_branch), classification reasoning (category + reason), and constraints ("only modify files related to X", "fix root causes")
    - Creates a Developer instance with profile's agent config, passing {"developer.system": PROMPT_DEFAULTS["developer.pr_fix.system"].content} as prompts dict to override the system prompt
    - Creates a temporary ImplementationState with goal=goal_text, plan_markdown=goal_text (required by Developer._build_prompt)
    - Calls Developer.run() and iterates through the async generator to completion
    - On success: appends GroupFixResult(file_path=path, status=FIXED, comment_ids=ids)
    - On exception: appends GroupFixResult(file_path=path, status=FAILED, error=str(e), comment_ids=ids), logs error, continues
    - Returns dict with group_results (list of all GroupFixResult) and agentic_status (COMPLETED or FAILED if all groups failed)
    - If file_groups is empty: returns empty group_results and status COMPLETED (graceful zero-actionable path)
  </behavior>
