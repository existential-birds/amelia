<objective>
Implement `reply_resolve_node` -- the final node in the PR auto-fix pipeline that closes the feedback loop by replying to reviewers and resolving threads.

Purpose: After fixes are pushed, reviewers need notification of what changed (or why a fix failed). This completes the autonomous fix cycle: detect -> classify -> fix -> push -> reply -> resolve.

Output: Working `reply_resolve_node` with per-comment replies, conditional thread resolution, error isolation, and full test coverage. Graph updated to: classify -> develop -> commit_push -> reply_resolve -> END.

Note: PIPE-08 (review pipeline composition) is explicitly deferred per user decision -- no PR creation capability exists yet. A separate plan (05-02) documents this deferral.
