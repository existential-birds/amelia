# Spec Builder Option Text Overflow Fix Implementation Plan

**Goal:** Fix multiple choice option buttons in the spec builder chat so long text wraps within the message bubble instead of overflowing.

**Architecture:** The base `Button` component applies `whitespace-nowrap` via `button-variants.ts`. The fix overrides this on option buttons in `AskUserQuestionCard.tsx` with `whitespace-normal text-left max-w-full break-words`, keeping the global button behavior unchanged.

**Tech Stack:** React, Tailwind CSS, Vitest, React Testing Library

## Phase 2: Apply the Fix

### Task 2: Add wrapping classes to option buttons in AskUserQuestionCard

**Files:**
- Modify: `dashboard/src/components/ai-elements/AskUserQuestionCard.tsx:108`

**Step 1: Update the option Button className**

In `AskUserQuestionCard.tsx`, line 108, change:

```tsx
className="flex flex-col items-start h-auto py-2 px-3"
```

to:

```tsx
className="flex flex-col items-start h-auto py-2 px-3 whitespace-normal text-left max-w-full break-words"
```

This adds four classes:
- `whitespace-normal` — overrides the base button's `whitespace-nowrap` so text wraps
- `text-left` — ensures wrapped text aligns left (the base button uses `justify-center`)
- `max-w-full` — prevents the button from growing beyond its parent container
- `break-words` — handles edge cases with very long unbreakable strings

**Step 2: Run the test to verify it passes**

Run: `cd dashboard && pnpm vitest run src/components/ai-elements/__tests__/AskUserQuestionCard.test.tsx`

Expected: ALL PASS — including the new `whitespace-normal` test.

## Summary

This is a minimal, targeted CSS fix. The root cause is `whitespace-nowrap` in the base `Button` variant (`button-variants.ts:3`). Rather than modifying the global button styles (which could have unintended side effects), we override on the specific option buttons in `AskUserQuestionCard.tsx` with `whitespace-normal text-left max-w-full break-words`. One new test validates the fix. The change touches exactly one production file and one test file.
