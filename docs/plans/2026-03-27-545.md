# Remove Activity Log Implementation Plan

**Goal:** Remove the activity log section from the workflow detail page and workflows page, deleting all related components and cleaning up unused dependencies.

**Architecture:** The activity log is rendered in a right column on WorkflowDetailPage and a right panel on WorkflowsPage. Removal involves deleting the component tree (ActivityLog + activity/ subdirectory), adjusting both page layouts to single-column, removing the `recent_events` field from the API type and all consumers, cleaning up event-merging logic, and deleting associated tests. The `@tanstack/react-virtual` dependency is retained because LogsPage still uses it.

**Tech Stack:** React, TypeScript, Vitest, TanStack React Virtual

## Summary

This plan removes the activity log feature in 5 phases:

1. **Phase 1** (Tasks 1-2): Remove ActivityLog from WorkflowDetailPage, adjust layout to single-column, update tests
2. **Phase 2** (Tasks 3-4): Remove ActivityLog from WorkflowsPage, adjust layout, update tests
3. **Phase 3** (Task 5): Delete all ActivityLog component files and the `activity/` directory (11 files total)
4. **Phase 4** (Tasks 6-9): Remove `recent_events` from the WorkflowDetail type, update `getEndTime` utility, clean up fixtures and mock data
5. **Phase 5** (Tasks 10-11): Full verification and commit

**Key decisions:**
- `@tanstack/react-virtual` is **retained** because `LogsPage.tsx` still uses it
- `useWorkflowStore` is **retained** on WorkflowDetailPage for recovery detection (checking `workflow_failed` events with `recoverable: true`)
- The WebSocket event subscription infrastructure is **untouched** since it serves other features beyond the activity log
- CSS agent color variables in `globals.css` and `AGENT_STYLES` in `constants.ts` are **retained** since they're used by other components (UsageCard, LogsPage)
